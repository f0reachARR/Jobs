# 利用者ガイド（基本機能）

このガイドは Jobs プラグインが導入されたサーバで遊ぶプレイヤー向けに、ふだん使う機能をまとめたもの。
発展的な仕組み（単調性ペナルティ、自動化対策など）は [player-advanced.md](./player-advanced.md) を読む。

## Jobs プラグインとは

このプラグインでは、プレイヤーは **1 つの職業（＝専業）** を選び、その職業に対応する行動をとると報酬（通貨）がもらえる。
たとえば「討伐」職を選んだプレイヤーが敵 mob を倒すと報酬がもらえるが、鉱石を掘っても報酬にはならない。

- **専業は 1 つだけ**：同時に複数の職業に就くことはできない。
- **専業外の行動には報酬もログもない**：他の職業のアクションをしても、報酬は出ず、記録も残らない。
- **変更にはクールダウン**：一度選んだ専業を変えるには時間を置く必要がある。

## 最初にすること：専業を選ぶ

初回ログイン時、専業選択ダイアログが自動で表示される（サーバ設定次第）。
表示されなかった場合や、後から選び直したい場合は次のコマンドを使う。

```
/jobs select
```

- **未選択のとき**：職業一覧のダイアログが開く。
- **選択済み・変更可能なとき**：職業変更のダイアログが開く。
- **選択済み・クールダウン中のとき**：次回変更可能な時刻を示すダイアログが開く。

一覧から職業ボタンを押すと、その職業の詳細ダイアログが開く。
そこで報酬対象や条件を確認したうえで「この職業を選ぶ」を押すと確定する。

**注意**：ダイアログを閉じただけでは何も選ばれない。専業未選択の間はいくら行動しても報酬もログも発生しないので、`/jobs select` で再度開いて確定する。

## 職業の内容を確認する

自分がまだどれを選ぶか決めていない、あるいは他の職業がどんな報酬を出すのか見たいときは次を使う。

```
/jobs info              職業一覧を開く
/jobs info <job-id>     特定の職業の条件だけを開く
```

このコマンドは選択状態やクールダウンとは関係なく、いつでも実行できる。
`<job-id>` はサーバに配置されている職業 YAML の `id` に相当する。タブ補完はされないので、一覧から開くほうが確実。

詳細ダイアログでは以下のような情報が並ぶ。

- 職業表示名と簡単な説明
- 報酬対象アクションの一覧（例：「討伐: minecraft:zombie」「採掘: #minecraft:ores」）
- 報酬額（サーバ設定で伏せる運用にしていることもある）
- 単調性ペナルティが有効かどうか、その概要
- 自動化対策として何が有効になっているか

サーバ運営がイベントを組んでいる場合、ここに書かれた対象・額が「今このサーバで稼げる範囲」の一次情報になる。

## 自分の状態を確認する：`/jobs status`

現在の専業や本日の獲得額を知りたいときは次を使う。

```
/jobs status
```

チャット欄に次のような情報が並ぶ。

- **現在の専業**
- **本日の獲得額と上限**：`amount / cap` の形式。上限（日次キャップ）に達すると以降その日は報酬 0 になる。
- **進捗バー**：`[■■■■□□□□□□]` のような表示で、上限までの進捗を可視化する。
- **単調性ペナルティの状態**：作動していれば注意メッセージが出る。詳しくは [player-advanced.md](./player-advanced.md)。
- **次回変更可能時刻**：クールダウンが残っていれば `yyyy-MM-dd HH:mm` 形式で表示される。

専業未選択のとき、または日次キャップの上限が 0 になっている場合、対応する行は表示されない。

## 専業の変更

一度選んだ専業を変えるには、変更クールダウンが切れている必要がある。
`/jobs select` を実行すると、変更可能なら一覧ダイアログが、まだクールダウン中なら「あと何分」を示すダイアログが開く。

- **変更中の職業を除外**：変更ダイアログには現在の職業は表示されない。
- **同じ職業を再確定するとどうなるか**：技術的には可能だが「変わっていない」旨のメッセージが返るだけでクールダウンは消費されない。
- **クールダウンの長さはサーバ次第**：イベント期間中は短く、通常期は数日、といった運用が多い。詳しくは `/jobs status` の「次回変更可能」で自分に適用される時刻を確認する。

## `/jobs` コマンドまとめ

利用者が使うコマンドはこれだけ。

| コマンド | 用途 |
|---|---|
| `/jobs` | 引数なし。`/jobs select` と同じ挙動。 |
| `/jobs select` | 専業の選択・変更。状況に応じて 3 種類のダイアログに分岐。 |
| `/jobs info` | 職業一覧を閲覧（読むだけ）。 |
| `/jobs info <job-id>` | 特定職業の詳細を閲覧。 |
| `/jobs status` | 自分の現状（専業・当日獲得・単調性・次回変更）を表示。 |

## よくある勘違い

- **「他の職業も少しはお金入ると思ってた」**：入らない。専業判定は完全に排他で、専業外のアクションはログにも残らない。
- **「専業なしでもとりあえずプレイしたい」**：報酬・行動ログどちらも発生しないため、Jobs プラグインの機能を使わない状態でプレイすることになる。
- **「/jobs change はどこ？」**：`/jobs change` は無い。選択も変更も `/jobs select` に統合されている。
- **「クールダウンが表示されているのに変更できた」**：管理者が `/jobs admin reset-cooldown` などで解除している可能性がある。あるいは `jobs.bypass.cooldown` 権限が付与されている（イベント時のスタッフなど）。

## Bedrock Edition プレイヤーへ

Bedrock 版（Geyser 経由）からも同じダイアログが使える。
ただし文字装飾（色・太字など）はクライアント側で無視されるので、Java 版と見た目が違っても内容は同じ。
一部の演出（自動で閉じるボタンなど）が動かないことがあるが、選択・確定ボタンは通常どおり使える。

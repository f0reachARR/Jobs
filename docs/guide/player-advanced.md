# 利用者ガイド（発展機能）

このガイドは Jobs プラグインの「仕組み」を深く知りたいプレイヤー向け。
コマンドの使い方や専業選びは [player-basic.md](./player-basic.md) を先に読む。

ここで扱うのは、ふだん意識しなくても遊べるが、把握しておくと稼ぎ方や動きに納得しやすくなる要素。

- 単調性ペナルティ（同じことばかりすると倍率が下がる）
- 自動化対策（スポナー・自動化装置由来の報酬は 0 になる）
- 日次キャップ（1 日あたりの報酬上限）
- rare 報酬（低確率で高額）
- 報酬パイプラインの流れと、なぜ「入らないアクション」が起きるか
- Bedrock 版特有の注意

## 単調性ペナルティ

同じ対象・同じ行動ばかりを繰り返すと、その職業で得られる報酬倍率が下がる仕組み。
たとえば「討伐」職でずっとゾンビだけ倒し続けると、直近 30 件中のゾンビ比率が上がって倍率が 0.75 → 0.50 と下がっていく。

### 挙動

- **対象**：職業ごとに設定される。すべての職業に付いているわけではないので、`/jobs info` で確認する。
- **単位**：報酬エントリ 1 件ごとに派生キー（`kill:minecraft:zombie` のような識別子）が振られていて、この派生キー単位でカウントする。OR リストやタグでまとめた報酬エントリは 1 バケットとしてカウントされるので、内訳を分散させたければ「別々の対象を対象にする報酬エントリ」を含む職業を選ぶ必要がある。
- **ウィンドウ**：直近何件分を対象にするかは職業ごとに違う（多くの場合 20〜30 件）。ログアウトすると memory 上のリングバッファはクリアされる（オフライン中に持ち越されない）。
- **発動条件**：バッファがウィンドウ件数に達するまでは発動しない。開始直後は倍率フル。
- **数値の非開示**：多くの職業は `hide_numbers: true` で設定されているため、`/jobs status` 上には「同じものばかり扱うと効率が落ちる」のような一言だけが出て、具体的な倍率や比率は隠されている（見せる運用にしているサーバでは数値が出る）。

### 対策

- 対象を分散させる。討伐なら複数種類の mob を混ぜる、採掘なら鉱石種類を回す。
- ペナルティは行動ログには影響せず、報酬の倍率だけ落とすので、単調性が出ていても「行動が無効化された」わけではない。
- ログアウト → 再ログインで memory 上のバッファがクリアされる。強制解除の手段としては使えるが、通常運用でこれを使う必要はない。

## 自動化対策

Jobs プラグインは、報酬パイプラインの中で「これは自然な労働ではない」と判定した行動を **報酬 0 に固定** する。
行動ログには残るが、通貨は入らない。

サーバ設定と職業設定に応じて、次のいずれか（または複数）が有効になっている場合がある。

| 種類 | 発動条件 | 対象アクション |
|---|---|---|
| `spawner_origin_kills` | スポナー由来の mob を倒したとき | `entity_killed` |
| `unplanted_crop_harvest` | 自分（または他プレイヤー）が植えていない作物を収穫したとき | `block_broken`（作物のみ） |
| `recently_placed_break` | 直近に置かれたブロックを破壊したとき | `block_broken`（作物以外） |
| `auto_fed_processing` | Furnace や BrewingStand に人手ではなくホッパー等で材料を入れたとき | `item_smelted`, `item_brewed` |
| `villager_repeat_trade` | 同じ村人と短時間に同じ取引を繰り返したとき | `villager_traded` |
| `breed_non_player_breeder` | プレイヤー以外（ディスペンサー等）の交配 | `entity_bred` |

### 発動時の通知

サーバ側で通知が有効になっている場合、報酬が 0 になったときにアクションバー（画面下部）で理由が表示される。
通知はデフォルトで on になっているが、reason ごとにサーバ側で無効化されていることもある。

### よくある混乱

- **「経験値は入ったのに通貨だけ入らない」**：Minecraft の経験値と Jobs の報酬は別系統。自動化対策が発動していると通貨だけ 0 になる。
- **「自分が植えた作物のはずなのに 0 だった」**：チャンクがアンロードされた／爆発で吹き飛んだブロックは「植えた」フラグが失われる仕様。
- **「置いてすぐ壊しただけで 0 になった」**：置いてから設定秒数（デフォルト 3600 秒 = 1 時間）以内の破壊は `recently_placed_break` の対象。置いて即回収は無効化される。
- **「Furnace の中身を取り出したのに 0」**：入れたのがホッパーだと `auto_fed_processing` で 0 になる。人手で入れれば通常どおり報酬対象。

## 日次キャップ

1 日（サーバ時間 00:00 リセット）に稼げる総額に上限がある。
サーバ設定で以下のどちらかの scope を採る。

- **`total`**：全職業合算の上限。専業を変えても累計は引き継がれる。
- **`per_job`**：職業別の上限。

上限に達すると、その日はその scope での報酬が 0 になる。行動ログには残る。
`/jobs status` の「本日の獲得額」に表示される進捗バーで到達度が分かる。

**リセットのタイミング**：デフォルトはサーバ時間の 00:00。ログアウトしていてもリセットは自動で走る。

## rare 報酬

一部の報酬エントリには `rare` が設定されていて、1 アクションあたり低確率（例：0.001 = 0.1%）でヒットすると通常報酬が高額に置き換わる。

- **通常報酬と rare 報酬は排他**：rare がヒットした瞬間、通常報酬は支払わずに rare の額で上書きされる。合算はされない。
- **`announce`**：rare がヒットしたときにサーバ全体にメッセージが流れる場合がある。演出の一部なので、遭遇するまでどんな条件かは意図的に伏せられている。
- **表示**：`/jobs info` の詳細ダイアログでは「レア報酬あり」の 1 行だけ出て、確率と額は伏せられている。

日次キャップは rare 込みで判定されるので、rare を引いた直後に上限を突破することがある。

## 報酬パイプラインの流れ

「何をしたら報酬になるか」の判定は、ざっくり次の順で走る。

1. **マッチング**：あなたのアクションが、専業 YAML の報酬エントリのどれと一致するかを判定。上から見て最初にマッチしたエントリだけが採用される（`first match wins`）。
2. **専業判定**：マッチしたエントリの職業と、あなたの専業が同じか。違えばここで完全終了（報酬もログもなし）。
3. **自動化対策**：スポナー由来などの条件に該当すれば報酬 0 に固定（ログは残る）。
4. **基礎報酬計算**：`reward` の値、または min/max の範囲乱数から額を出す。amount 系（smelt / craft / trade / brew）は個数倍。
5. **rare ロール**：低確率で高額に置換。
6. **内蔵倍率補正**：単調性ペナルティ → 日次キャップ。
7. **拡張倍率**：サーバがイベントプラグイン等で登録した倍率がここに入る。
8. **報酬の分配**：サーバがイベントプラグイン等で登録した分配ロジック（税、団体拠出など）がここに入る。
9. **丸め**：小数点以下を設定に従って丸める。
10. **送金・ログ書き込み**：ここで初めて通貨口座に入る。

「行動したのに報酬が入っていない」ケースは 1〜3 のどこかで止まっている可能性が高い。まず自分の専業がその報酬対象を持つジョブか確認する（`/jobs info` で対象一覧を見る）。

## 「入らないアクション」の切り分けチェック

順に確かめる。

1. **専業を選んでいるか**：`/jobs status` で確認。未選択なら何も入らない。
2. **その職業の報酬対象か**：`/jobs info <job>` の対象一覧に、行った行動（対象アイテム／mob）が入っているか。
3. **first match wins に負けていないか**：報酬エントリの順序次第で、期待したエントリより先に別のエントリが確定していることがある。サーバ運営に確認する。
4. **自動化対策に引っかかっていないか**：アクションバー通知が出ていないか。スポナー由来／直近に置いたブロック／ホッパー投入などに心当たりがないか。
5. **日次キャップに達していないか**：`/jobs status` の「本日の獲得額」を見る。
6. **単調性ペナルティは倍率下げるだけで 0 にはしない**：これが原因で「まったく入らない」ことはない。額が想定より少ない場合の要因。

## Bedrock Edition プレイヤー向けの注意

- ダイアログの装飾（色・太字）は Bedrock 側で表示されない。文言そのもので判断する。
- 一覧ダイアログの「一覧に戻る」は、Bedrock 側ではダイアログを閉じてから新しく一覧を開き直す形で実装されている。裏側では同じ機能。
- 一度開いたダイアログを外部から自動で閉じる操作は Bedrock では機能しないことがある。基本的にプレイヤー自身が確定ボタンか閉じるボタンを押すことで進む。

# ADR-0002 専業外アクションは報酬と記録の双方を行わない

## ステータス

受け入れ

## 背景

イベントルールブックは「専業アクションのみが報酬対象」と定めている。
これを Job プラグイン側で実装するにあたり、専業外アクションをどう扱うかには選択肢がある。

第一に、報酬は 0 にしつつ、行動ログには残す案。
業績指標や Quest の評価に「専業外でも何をしていたか」を使う余地ができる。

第二に、報酬もログも一切残さない案。
専業外アクションは Job プラグインから見て存在しないものとして扱う。

第一案は将来の拡張余地としては魅力的だが、副作用として「専業を変えていなくても専業外の活動が業績指標に乗る」可能性が出てくる。
業績指標を平等に扱うルールと衝突しかねない。
また、ログ書き込み量が単純に増え、自動化アタックの記録コストも増える（リスポーンと連打を 24 時間続けるような単純 bot が、報酬 0 でも MySQL に書き込み続ける）。

## 決定

第二案を採用する。

`config.yml` の `specialty_mode.reward_non_specialty` を 0.0 にする運用を既定とする。
このとき、報酬パイプラインの段階 2（専業判定）でプレイヤーの専業 ID と一致しないアクションは、報酬支払いを行わず、行動ログにも書かない。

将来「専業外も記録だけする」モードが必要になった場合は、`reward_non_specialty` を 0.0 のまま `log_non_specialty` フラグを別途追加するなどの拡張で対応する。
現時点では仕様に含めない。

## 結果

- 専業外の活動は Job プラグインから見えない。
  業績指標や Quest 評価も自然に専業内活動だけを扱う。
- 行動ログのサイズが「専業活動のみ」に絞られ、自動化アタックの書き込み圧迫を回避できる。
- 専業未選択のプレイヤーはどのアクションでも報酬が出ないため、初回専業選択ダイアログを早期に促す UX 設計が必要になる（[07-ui.md](../07-ui.md) を参照）。

## 選択しなかった代替案

「報酬 0、ログだけ残す」案は将来拡張で扱う余地を残すが、現時点での要件には含まれない。
専業の意味を「報酬対象の絞り込み」に明確化することで、ルールの予測可能性を高める。
